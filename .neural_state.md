## ⚠️ LECCIONES APRENDIDAS Y TRAMPAS (IMPORTANTE PARA OTRO AGENTE)

### 1. Deploy Edge Functions — Ruta problemática
El workspace está en OneDrive con caracteres especiales (acentos, espacios). Para deploy con CLI:
```bash
xcopy /E /I "supabase" "C:\temp\sb-deploy\supabase"
cd C:\temp\sb-deploy
supabase functions deploy NOMBRE --project-ref bhdictzvboiojyxorfiq --no-verify-jwt
```
Alternativa más simple: editar código directo en Supabase Dashboard → Edge Functions.

### 2. Edge Functions deben deployarse MANUALMENTE
Crear la función en el Dashboard genera un stub "Hello world". HAY QUE editar el código y pegar el real desde supabase/functions/NOMBRE/index.ts del repo.

### 3. JWT Verification debe estar OFF
En Dashboard → Edge Functions → cada función → Settings → "Verify JWT" = OFF. Si no, rechaza con "Invalid credentials". El storefront (sin auth) necesita llamarlas.

### 4. Gemini API key formato AQ.Ab8R...
AI Studio ahora genera tokens OAuth `AQ.Ab8R...` (NO el clásico `AIzaSy...`). Ambos funcionan pero SOLO con el header `x-goog-api-key` (no con `?key=` en URL). El Edge Function chat ya está adaptado.

### 5. SQL Migrations — Palabras reservadas como alias
En PostgreSQL, los alias de columnas en `VALUES ... AS v(col1, col2)` NO pueden ser palabras reservadas sin entrecomillar: `desc`, `key`, `name`, `time`, `path`, `level`, `sort`. Usar dobles comillas: `AS v("key", "name", "time")`. Y en el SELECT referenciar igual: `v."key"`, `v."name"`.

### 6. SQL — WITH CHECK no aplica a SELECT/DELETE
Las policies `FOR SELECT` solo usan `USING`, NUNCA `WITH CHECK`. Solo `FOR INSERT`/`FOR UPDATE`/`FOR ALL` admiten `WITH CHECK`.

### 7. SQL — Casts explícitos en VALUES con tipos mixtos
Cuando un `VALUES` mezcla TEXT, INTEGER y NUMERIC, PostgreSQL falla la inferencia. Usar casts: `1200::int`, `4.9::numeric`, `'texto'::text`.

### 8. GitHub Pages publica desde main, NO desde gh-pages
La config del repo tiene `build_type: legacy`, `source: branch main`. El workflow minify-deploy.yml existe pero NO se usa (intentaba publicar a gh-pages que no existe). Los cambios se publican automáticamente al push a main (sin minificar, pero funcionales).

### 9. Chart.js requiere contenedor con altura definida
Con `maintainAspectRatio: false`, Chart.js usa la altura del div padre. Si no tiene altura explícita, el canvas se renderiza con altura 0 → gráfico invisible. Solución: contenedor `.chart-box { height: 200px; position: relative; }`.

### 10. bullet Unicode (•) causa caracteres raros en WhatsApp
El caracter `•` (U+2022) al inicio de líneas del mensaje de orden se ve mal en algunos clientes WhatsApp. Usar guion simple `-` en su lugar.

### 11. Función is_admin() en SQL
```sql
CREATE OR REPLACE FUNCTION is_admin() RETURNS BOOLEAN LANGUAGE sql SECURITY DEFINER STABLE AS $$
  SELECT EXISTS (SELECT 1 FROM auth.users WHERE id = auth.uid()
    AND (email LIKE '%@telocg.com' OR raw_user_meta_data->>'role' = 'admin'));
$$;
```
Otorga permisos admin automáticamente a emails @telocg.com.
